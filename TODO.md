## Текущее состояние репозитория (снимок)

Уже есть **сквозной сценарий**: регистрация → тест (Likert / binary / forced_choice, биполярные подписи и текстовые подсказки по шагам) → summary → лента с `%` и инсайтами по осям → экран пользователя → сравнение → лайки → список диалогов и чат → **тематические обсуждения** по осям (`/discussions`).

- **Стек зафиксирован:** FastAPI + SQLAlchemy + Alembic, Next.js 14 + Tailwind, JWT, SQLite dev / PostgreSQL prod.
- **Сущности в БД:** `User`, `Question`, `QuestionAxis`, связи вопрос–ось, `Answer`, лайки, чаты/сообщения, **посты и комментарии обсуждений** (`discussion_posts`, `discussion_comments`, в т.ч. `reply_to_comment_id`), см. `backend/app/models/`.
- **Матчинг v0:** `backend/app/core/matching.py` — оси из ответов через `question_axis_link`; сравнение только по осям с данными у обоих; инсайты по порогам расстояния.
- **Сиды:** `backend/seed.py` — оси и онбординг; `backend/scripts/seed_fixture_users.py` — симуляция N пользователей (по умолчанию 100) со **случайными** ответами (`--count`); пароль демо см. `README.md` / константа `CHAOS_PASSWORD` в скрипте.
- **Фронт:** единая **дизайн-система** (`mm-*` в `frontend/app/globals.css`), Geist, нижняя навигация с активным разделом; опрос ленты обсуждений/комментариев с **адаптивным интервалом** при отсутствии новых данных; на чате/ленте/группе — **параллельная** загрузка независимых запросов где возможно (`Promise.all`).
- **Производительность API:** лента, групповой матчинг и блок «общее в комнате» используют **`compute_user_axis_scores_batch`** вместо N× полного пересчёта профиля в цикле (см. `README.md`, «Производительность и задержки»).
- **Пробелы относительно идеала MVP:** OAuth, деплой и метрики, второй пак вопросов, PWA — см. фазы ниже.

## Фаза D — качество и запуск

- [ ] **Доступность:** не полагаться только на цвет (частично: подписи ✓/≠ в ленте); аудит контраста — по желанию; **фокус/лейблы/sr-only** после дизайн-обновления — summary + обсуждения (итерация).
- [x] **Сиды для демо:** `seed_fixture_users.py` (массовая симуляция со случайными ответами).
- [ ] **Деплой:** staging + прод; HTTPS; бэкапы БД.
- [ ] **Метрики:** завершили тест, лента, лайк, сообщение.
- [x] **Фронт:** `next/image` (аватары ленты, обложки/превью обсуждений, `unoptimized` где нужен произвольный origin); **ETag + If-None-Match** для опроса списка постов и комментариев (304 при отсутствии изменений).
